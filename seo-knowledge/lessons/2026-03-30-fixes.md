# 2026-03-30 Fixes

Summary of SEO and build fixes applied on March 30, 2026.

---

## 1. Service Schema: Added Missing Fields

**Problem**: K金回收, 白金回收, and 鑽石回收 service pages had incomplete `Service` schema — missing `offers`, `url`, and `areaServed`.

**Fix**: Added the following to each service schema:
- `offers` with `@type: Offer`, `price`, `priceCurrency: TWD`
- `url` pointing to each service's canonical page
- `areaServed` with `AdministrativeArea` entries for Taipei/New Taipei

**Takeaway**: Every `Service` schema must include `offers`, `url`, and `areaServed` to qualify for rich results. Use the structured-data checklist before adding new service pages.

---

## 2. Organization postalCode Unified to 10647

**Problem**: `postalCode` was inconsistent across schema instances — some pages used `106`, others `10647`.

**Fix**: Unified all Organization and JewelryStore schema `postalCode` to `10647` (full 5-digit Taiwan postal code).

**Takeaway**: Always use the full 5-digit postal code for Taiwan addresses. Add this to the NAP consistency audit checklist. The canonical postal code is **10647**.

---

## 3. robots.txt: Blocked CCBot and Bytespider

**Problem**: AI crawlers (CCBot from Common Crawl / Anthropic training, Bytespider from ByteDance) were indexing the site without adding search value.

**Fix**: Added the following to `robots.txt`:
```
User-agent: CCBot
Disallow: /

User-agent: Bytespider
Disallow: /
```

**Takeaway**: Regularly review which bots are crawling the site (check server logs or Vercel analytics). Block crawlers that consume resources without driving traffic. Keep a running list of blocked bots in `robots.txt` with comments explaining why.

---

## 4. Vercel Preview URL Redirect

**Problem**: Vercel preview deployments (e.g., `gold-web-v2-xxx.vercel.app`) were being indexed by Google, creating duplicate content noise in Search Console. Previous approach used `noindex` meta tags, but preview URLs still appeared in Search Console reports.

**Fix**: Changed strategy from `noindex` to **301 redirect**. Preview URLs now redirect to `https://www.gold-tw.com` via Vercel middleware or config. This completely prevents Search Console from registering preview URLs.

**Takeaway**: For Vercel projects, redirecting preview URLs to production is more effective than `noindex`. The redirect prevents Search Console from ever seeing the preview URL as a separate page. Implement this early in any new Vercel project.

---

## 5. JSX `>` Symbol Must Be Escaped

**Problem**: Raw `>` character in JSX text content caused a Next.js build error. The build failed silently in dev mode but broke in production build.

**Fix**: Replaced `>` with `&gt;` in JSX content (line 184 of the affected file).

**Takeaway**: In JSX, always use `&gt;` for the `>` symbol in text content. The `<` and `>` characters have special meaning in JSX. **Always run `next build` locally before pushing** to catch these syntax errors early.

---

## Action Items / Lessons Learned

1. **Pre-push checklist**: Always run `next build` locally before pushing. JSX syntax errors may not appear in `next dev` but will fail the production build.
2. **Schema completeness**: Use the structured-data checklist (`seo-knowledge/core/structured-data.md`) when adding or modifying any schema. Every required field must be present.
3. **NAP audit**: When changing any address field, search the entire codebase for the old value and update all instances. Use: `grep -r "postalCode" --include="*.tsx" --include="*.ts"`.
4. **robots.txt maintenance**: Review and update bot blocks quarterly. Document each block with a comment.
5. **Preview URL handling**: Implement production redirects for preview URLs on day one of any new Vercel project.
